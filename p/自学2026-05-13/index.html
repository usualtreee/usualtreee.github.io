C20 Test 3 Passage 1: Frozen Food 学习笔记

标题与核心词汇

  • frozen (freeze 的过去分词) = 冷冻的；结冰的
  • frozen food = 冷冻食品 / 速冻食品

第一段：古代冷冻技术

  • inhabitants = 居民；居住者；栖息动物
    • 词根：in-（在…里）+ habitare（居住）+ -ant（人）
    • 同义替换：residents / dwellers / population
    • 与 habitat（栖息地，地方）同源，都来自拉丁语 habitare
    • 与 habit（习惯）也有远亲关系，都来自"保持、居住"的概念
  • trampled (trample 的过去分词) = 被踩踏的；被踩坏的
  • nutritional (nutrition + -al) = 营养的；与营养相关的
    • 常见搭配：nutritional value（营养价值）/ nutritional loss（营养流失）
    • 区分：nutrition（营养，抽象）/ nutrient（营养素，具体）/ nutritious（有营养的）
  • aesthetic appeal = 审美吸引力；美观度；视觉吸引力
    • aesthetic = 美学的、审美的
    • appeal = 吸引力
    • 同义替换：visual appeal / attractiveness / appearance

第二段：冰块运输与机械制冷

  • towed (tow 的过去式/过去分词) = 被拖运的；被牵引的
    • 原文：ship owners… had enormous blocks of Arctic ice towed all over the Atlantic
    • 语法：have sth done（让某事被做）
    • 船主安排船只把巨大冰块拖运到大西洋各地
  • enormous = 巨大的；庞大的（强调尺寸/体积大，不是数量多）
    • ❌ 不等于 many / numerous
    • ✅ 同义：huge / gigantic / massive
  • butter（黄油） vs butterfly（蝴蝶）= 拼写巧合，现代英语中无实际关系
  • compressor = 压缩机（压缩气体，使其高压高温）
  • condenser = 冷凝器（让热气体散热，凝结成液体）
    • condense = （气体）凝结成液体；（内容）浓缩
    • 区分：compress（压缩，还是气体）→ condense（冷凝，变成液体）
  • evaporator = 蒸发器（液体快速蒸发，吸热变冷）

第三段：Birdseye 与速冻技术

  • mutton = 羊肉（成年绵羊的肉，区别于 lamb 羔羊肉）
  • palatable = 美味的；可口的
    • 引申义：可接受的、合意的（如 proposal was not palatable）
  • deterioration (deteriorate 的名词) = 恶化；变质；退化
  • burst = 爆裂；破裂
    • 冷冻语境：水结冰膨胀导致细胞 burst（破裂），影响口感
  • spoilt（英式）/ spoiled（美式）= 变质的；腐坏的；被宠坏的
  • indigenous = 本土的；原产的；土著的
    • 同义替换：native / local / original
  • ran out of = 用完了；耗尽了（run out of 的过去式）
    • 主语是人：We ran out of ice.
    • 区分：run out（主语是物）The ice ran out.

第四段：包装与市场推广

  • as well as = 也；和……一样；除……之外还
    • ⚠️ 主谓一致：谓语动词跟 as well as 前面的词一致
    • 强调前面的，后面是附加信息
    • 原文连接两个 technique：quick-freezing techniques as well as the technique of freezing…
  • cellophane = 玻璃纸；赛璐玢（透明包装材料）
    • 原文：the first transparent material for food packaging
  • came in = 以……形式/规格出售
    • 原文：Birdseye products also came in convenient size packages…
    • = 产品以方便规格推出
  • with a minimum of effort = 只需简单准备；最省事
  • grocery = 食品杂货；杂货店
    • grocery stores = 食品杂货店
    • 与足球词汇无关（注意区分 goalie / grassroots 等发音相近词）

第五段：二战与 TV Dinner

  • munitions = 军火；弹药；军需品
  • ration = 配给；定量供应（动词/名词）
    • 原文：Canned foods were rationed…
  • tin = 锡；马口铁（制造罐头和弹药的金属原材料）
  • compartment = 隔间；舱室；分隔空间（如冰箱冷冻舱、火车车厢）
  • TV-dinner = 电视餐；速冻即食餐（托盘装，可边看电视边吃）
  • trusted brand name = 值得信赖的品牌；可靠的品牌名称
  • poultry = 家禽；家禽肉（鸡、鸭、鹅、火鸡等，区别于 livestock 牲畜）

第六段：Swanson 与 TV Dinner 成功

  • clever name = 巧妙的名称
    • 指"TV Dinner"蹭了当时电视普及的热度，起名很聪明
  • segmented = 分格的；分隔的
    • segmented aluminium tray = 分格铝制托盘（像飞机餐一样，一格一格分开装不同食物）

关键长难句翻译参考

  1. He developed quick-freezing techniques that reduced the damage that crystals caused, as well as the technique of freezing the product in the package it was to be sold in. → 他开发了能减少冰晶损害的速冻技术，以及将产品直接在其销售包装中进行冷冻的技术。

  2. Birdseye products also came in convenient size packages that could be prepared with a minimum of effort. → Birdseye 的产品还采用方便规格的包装，只需简单准备即可食用。

  3. In 1954, the company adapted some of Birdseye’s freezing techniques, and with the help of a clever name and a huge advertising budget, it launched the first ‘TV Dinner’. → 1954年，该公司采用了 Birdseye 的一些冷冻技术，并借助一个巧妙的名称和巨额广告预算，推出了第一款"电视晚餐"。

  4. This consisted of frozen turkey, potatoes and vegetables served in the same segmented aluminium tray that was used by airlines. → 这款电视晚餐由冷冻火鸡肉、土豆和蔬菜组成，盛放在与航空公司使用的相同的分格铝制托盘之中。

二、技术问答速查

问题 答案
FFmpeg是Java的吗？ 不是，C语言写的。Java项目通过命令行或JNI调用
Android是Java的吗？ 应用层是Kotlin/Java，系统底层是C/C++，内核是Linux（C）
iOS是基于Linux吗？ 不是，内核是XNU（Mach+BSD），和Linux是两条路
Demuxer/Muxer Demux=拆包（把容器里的音视频流分离），Mux=打包（把多路流合成一个文件）
SDL是什么 Simple DirectMedia Layer，负责音视频解码后的播放和渲染输出
RKNN Toolkit2 vs Lite2 Toolkit2在PC上做模型转换+量化；Lite2在板子上做推理（Python封装）
RKNN Runtime librknnrt.so，板端真正执行推理的C++运行时库
NPU是华为的吗？ 不是，NPU是通用术语。瑞芯微NPU（RK3588）定位嵌入式低功耗，华为昇腾定位云端/高端
RKNN支持ONNX为何还要转换？ ONNX是通用格式，NPU只认.rknn。转换过程做量化、算子适配、图优化
RK3588 6 TOPS怎么算 1024 MAC/cycle × 3核 × 1GHz × 2（1MAC=2次运算）≈ 6 TOPS
Jetson Orin Nano Super 官方超频版，硬件不变，软件解锁更高频率，67 TOPS，$249
RKNN模拟器 vs 连板 模拟器是Toolkit2内置的调试工具（PC端）；Runtime只能连真板运行

三、学习路径建议

已有 RTX 2060，先别急着买板子

  • 2060足够学会80%的模型部署技能：ONNX导出、TensorRT转换、INT8量化、推理优化
  • 等要投简历前12个月，再买 Jetson Orin Nano Super（¥20702300）做端侧迁移验证

RKNN vs Jetson 优先级

  • 先死磕Jetson + TensorRT（行业通用，简历值钱）
  • RKNN作为补充：花3~5天快速跑通转换和部署流程，简历多一条"熟悉国产NPU"

端侧 vs 云端 AI Infra 技术栈对比

层级 端侧（RK/Jetson） 云端（A100/H100）
推理框架 RKNN Runtime、TFLite、ONNX Runtime ARM、MNN、NCNN TensorRT、vLLM、Triton Inference Server、ONNX Runtime CUDA
编译优化 RKNN Toolkit2、INT8量化、算子回退 TensorRT Builder、XLA、TVM、Layer Fusion、Kernel Auto-Tuning
算子开发 Runtime Custom Op C++插件、CPU回退实现 CUDA Kernel、TensorRT Plugin、Triton Kernel
部署形态 交叉编译、固件烧录、OTA Docker + K8s、微服务、Serverless
硬件抽象 librknnrt.so、NPU驱动、OpenCL/Vulkan CUDA Runtime、cuDNN、cuBLAS、NCCL、NVLink
监控指标 Latency、FPS、CPU/NPU占用、温度、功耗(mW) QPS、TTFT、TPOT、GPU利用率、显存占用
核心矛盾 latency vs 功耗 vs 内存 吞吐量 vs 成本

四、工程思维：从零做项目

工程项目 ≠ 手敲每一行代码

真实项目是**“站在巨人肩膀上搭积木”**：

  1. 需求分析 → 2. 技术选型（clone LVGL、FFmpeg等库） → 3. 搭骨架（写main.c胶水代码）
  2. 移植集成（交叉编译、解决链接/内存问题） → 5. 增量开发（业务逻辑）
  3. 调试优化（不同步、卡顿、内存泄漏）

你真正需要手写的

  • 主框架和状态机
  • 模块间胶水层（UI点击怎么触发解码）
  • 错误处理和边界条件

建议补一个"从零开始"的项目

RK3588 智能相机/检测终端：

  • V4L2取流 → RKNN跑YOLO → LVGL显示结果框
  • 重点练：多线程同步、内存零拷贝、端侧完整pipeline
  • 写在简历上比"copy播放器"更能打动面试官
