RK3568 Linux 嵌入式开发今日知识点总结

一、编译体系：全量编译与单独编译

1.1 全量编译

./build.sh all

  • SDK 初次解压初始化必须完整执行一次
  • 自动编译uboot、内核、设备树、rootfs、recovery全部模块，生成完整可烧录整机系统
  • 编译耗时长，后续开发调试禁止重复执行

1.2 单独编译（改哪编哪，增量编译）

./build.sh uboot

  • 单独编译引导程序uboot

./build.sh kernel

  • 单独编译Linux内核 + 设备树

./build.sh rootfs

  • 编译buildroot极简根文件系统

./build.sh debian

  • 编译debian发行版根文件系统（新手首选）

./build.sh yocto

  • 编译yocto企业级定制根文件系统

./build.sh recovery

  • 单独编译系统急救分区recovery

1.3 编译后固件打包命令

./build.sh firmware

  • 将单独编译的模块更新到rockdev分区镜像目录

./build.sh updateimg

  • 打包所有分区，生成整机一键烧录镜像update.img

1.4 板级硬件配置

./build.sh BoardConfig-rk3568-evb1-ddr4-v10.mk

  • 作用：加载RK3568开发板专属硬件参数、分区配置
  • 规则：只要执行过一次./build.sh all，配置会永久保存，后续单独编译无需重复执行

二、核心镜像文件区分

2.1 uboot.img 与 boot.img 本质区别

  • uboot.img：开发板上电首个运行程序，负责底层硬件初始化、DDR配置、引导内核，不包含Linux系统本体
  • boot.img：由内核kernel.img+设备树resource.img打包而成，是Linux系统运行核心，由uboot负责加载
  • 二者为完全独立的硬件分区，互不包含

2.2 设备树 Device Tree

  • 定义：开发板专属硬件说明书，描述板子引脚、外设、硬件参数信息
  • 归属：设备树属于Linux内核的一部分
  • 编译规则：修改设备树.dts源码后，执行./build.sh kernel即可自动编译，新文件会打包进boot.img

2.3 根文件系统 rootfs

  • 定义：rootfs是根文件系统统称，是Linux用户运行区域，存放系统命令、软件、库、用户文件
  • 内核启动完成后挂载rootfs，Linux系统才算完整可用
  • 三种制作方案
    • buildroot：极简轻量嵌入式系统，无原生包管理器，适合产品量产
    • debian：新手首选，自带apt包管理器，软件生态完善，支持图形桌面
    • yocto：企业级定制框架，定制自由度高，入门开发无需接触

2.4 Recovery 分区作用

  • 定位：独立于主系统的系统急救小系统
  • 功能：系统救砖、固件升级、恢复出厂、分区格式化
  • 使用场景：正常开机不运行，仅主系统损坏时使用，日常开发基本用不到

三、开发板上电启动流程

  • uboot.img完成硬件初始化引导
  • 加载boot.img，内核解析设备树识别硬件
  • 内核加载外设驱动
  • 挂载rootfs根文件系统
  • 进入完整可用的Linux用户系统

四、拓展概念辨析

4.1 apt 与 pip 区别

  • apt：Debian/Ubuntu系统级包管理器，用于安装系统工具、编译器、底层软件
  • pip：Python专属包管理器，仅安装Python第三方库，无法管理系统软件
  • 二者功能互补，互不替代

4.2 系统环境说明

  • Windows原生不自带apt，apt仅存在于原生Linux、WSL、Linux虚拟机环境

五、开发调试基本原则

  1. SDK首次初始化，必须完整执行一次./build.sh all
  2. 后续开发严格遵守改哪编哪，不再重复执行全量编译
  3. 顶层build.sh脚本自动封装编译环境，仅手动进入源码目录时需手动配置ARCH变量
  4. 单独编译模块后，需执行./build.sh firmware更新镜像，才可烧录验证修改
