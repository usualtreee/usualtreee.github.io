雅思听力核心知识点总结

一、雅思听力口音核心规则

  1. 核心主体：80%为标准英式RP英音（BBC英音），是所有Section4学术讲座、核心独白的基准口音
  2. 次要口音：10%-15%为澳式/新西兰英语，仅5%以内为美式英语+其他轻口音
  3. 备考重点：无需专门练习美音，优先吃透剑桥真题英音材料；美式拼写（colour/color）官方均给分，不影响评分
  4. 考点提示：口音从来不是雅思听力考点，同义替换、定位词、细节拼写才是核心

二、高频方位短语：leading off

核心释义（按雅思出现频率排序）

  1. 【地图题必考】（道路/空间）从…延伸而出、与…相连、从…分岔开（以主路/主空间为起点向外分出支路）
    • 雅思真题例句：A narrow path leading off the main trail leads to the old quarry.
    • 核心辨析：leading off A=以A为起点分岔；leading to B=以B为终点到达
  2. 【独白/学术讨论高频】 开启、带头做、以…开场
    • 雅思真题例句：I’ll lead off with a brief overview of the project.

三、考古场景核心词：post holes & post

1. post holes

  • 正确拼写：post holes（复数，名词作定语必须用单数post，写posts hole直接零分）
  • 核心释义：考古场景专业术语「柱洞」，古代木质建筑的木柱埋入地下后留下的坑洞遗迹，是还原建筑形制的核心考古证据
  • 精听陷阱：音频中存在失爆连读，听起来像pos holes，极易漏听/t/、误写复数
  • 同义替换：structural remains of ancient buildings / foundations of the original building

2. post 核心义：柱子、桩、杆

  • 音标：英音 /pəʊst/
  • 雅思高频场景&搭配：
    搭配 含义 高频场景
    sign post 路标 地图题超高频定位词
    gate post 门柱 户外/景区场景
    fence post 围栏桩 生活/户外场景
  • 易混词辨析：
    单词 核心特征 雅思场景
    post 小型功能性垂直杆/柱，木质/金属，支撑/标识用 全场景最高频
    pole 细长无承重杆，仅作延伸/架设用 户外场景
    pillar 大型承重石柱/水泥柱，用于大型建筑 建筑/历史讲座

今日嵌入式Linux驱动学习总结

一、CPU架构区分

  1. ARM32：32位精简指令集，老旧嵌入式平台使用。
  2. ARM64(AArch64)：64位ARM，RK3568开发板架构，目前嵌入式主流。
  3. MIPS：老式精简指令架构，现已淘汰，新项目基本不用。 三者均为RISC架构，和电脑x86架构不通用，驱动程序必须交叉编译。

二、内核配置相关

  1. make menuconfig：内核图形化配置菜单，用来开关内核功能、设置驱动编译方式(y/M/)。
  2. Kconfig：菜单定义文件，编写menuconfig里的选项、依赖关系。入门外部驱动不用学，内置驱动才需要。
  3. .config：配置结果文件，menuconfig操作后自动生成，存放所有CONFIG_xxx配置。
  4. Makefile：编译规则文件。
    • 外部驱动只需背会极简版模板即可，不用深入复杂语法；
    • 三者关系：Kconfig定义菜单 → menuconfig生成.config → Makefile读取.config完成编译。

三、RKDevTool烧录文件区分

  1. boot.img：单分区启动镜像，仅含内核+设备树；单独烧录快，不改动根文件系统与用户数据，日常开发高频使用。
  2. update.img：全系统完整固件包，包含uboot、boot、rootfs所有分区；用于救砖、全新烧录整机系统。
  3. config.cfg(分区表)：固定文件，定义eMMC分区地址大小；和内核编译、驱动添加无关，全程用原厂固定配置即可。

四、Linux文件权限宏

S_IRUGO 代表所有用户只读权限，对应数字444，模块参数权限宏，定义sysfs节点访问权限。

五、内核模块 module_param 参数传递

  1. .ko = Linux内核模块，运行在内核态。
  2. insmod：加载模块命令，模块参数就在insmod加载时传入。
  3. module_param：普通整型参数加载传值。
  4. module_param_array：数组参数，array_size由内核自动统计传入元素个数，无需手动赋值。
  5. rmmod：卸载模块，触发module_exit函数。

六、内核符号表

内核所有函数、全局变量的地址+名称清单； 作用：insmod加载驱动时，靠符号表绑定内核函数地址，驱动才能调用printk等内核接口；未导出符号无法使用。

七、V4L2框架

Linux专属摄像头/视频采集通用框架，属于后期进阶内容； 前置基础极多（字符设备、I2C、platform总线等），现阶段完全不用学习，原厂多媒体驱动岗重点内容。

八、芯片原厂BSP岗位学习范围总结

前期：ARM架构、Uboot移植、DDR时钟、设备树DTS、内核基础子系统、存储(eMMC)、内核调试； 中期：各类总线驱动、电源管理； 后期：DRM显示、V4L2音视频、USB网络、内核补丁维护、固件量产打包。
