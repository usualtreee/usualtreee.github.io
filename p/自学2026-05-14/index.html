二、CUDA 与底层软件栈

CUDA 本质

  • 不是独立语言，是C/C++的GPU扩展方言，通过nvcc编译。
  • CUDA Kernel：__global__标记的函数，由CPU通过&lt;&lt;&lt;grid, block&gt;&gt;&gt;启动，在GPU上并行执行。
  • &lt;&lt;&lt;2, 2&gt;&gt;&gt;表示2个block×2个thread，共4个线程并行。

驱动与库的分层

层级 Linux Windows 谁写的
内核驱动 .ko (nvidia.ko) .sys GPU驱动工程师
用户态Driver API libcuda.so nvcuda.dll CUDA驱动团队
用户态Runtime libcudart.so cudart64_xxx.dll CUDA Runtime团队
推理框架 libnvinfer.so nvinfer.dll TensorRT/AI Infra团队
你的Plugin libplugin.so .dll 你（算法部署工程师）
  • .so：用户态共享库，封装函数/变量/数据，通过系统调用(ioctl/mmap)间接访问内核驱动，不能直接调用内核函数。
  • .ko：内核模块，直接操作硬件寄存器，用insmod加载。

嵌入式AI Infra优化五大方向（量化之后的进阶）

  1. 零拷贝（Zero-Copy）：MPP解码的MppFrame通过dma_buf直接共享给RKNN输入， eliminate memcpy。RK3588是UMA架构，CPU/NPU共享物理内存。
  2. 内存池（Memory Pool）：预先rknn_create_mem申请固定buffer循环复用，避免每帧malloc/free的碎片和锁竞争。配合双缓冲/三缓冲。
  3. 多线程流水线（Pipeline）：解码→预处理→推理→后处理四阶段用生产者-消费者模型并行。用std::mutex+condition_variable或原子无锁队列。绑核到big.LITTLE特定核心。
  4. RGA硬件加速：用librga.so替代OpenCV CPU做YUV→RGB转换和resize，速度提升5~10倍。
  5. 异步推理+Batch策略：rknn_run异步化（或双线程模拟），4路同分辨率可尝试Batch=4推理提升NPU利用率。

四、量化深入理解

核心概念

  • 量化：把FP32浮点数映射到INT8/INT16整数，缩小模型体积、加速推理、降低功耗。
  • PTQ（训练后量化）：模型训练完后用校准数据集统计范围，直接转INT8。边缘部署最常用，不需要训练环境。
  • QAT（量化感知训练）：训练时模拟INT8前向传播，精度更高但需重新训练。边缘团队通常没条件做。
  • 对称/非对称：对称量化zero_point=0，硬件更友好；非对称有偏移，精度略高。
  • per-tensor / per-channel / per-layer：全局一个scale太粗暴，按通道/按层独立统计精度更好。

量化调试

  • 精度掉了怎么办：逐层对比FP32 vs INT8输出、换校准数据集、用per-channel、敏感层保留FP16（混合精度）。
  • 和金融量化完全无关：金融量化是数学建模投资，模型量化是数值压缩。

五、硬件架构认知

NPU vs GPU

  • GPU：通用并行处理器（SIMT），CUDA Core/Tensor Core，完全可编程，功耗高（几百瓦）。
  • NPU：专用DSA（领域专用架构），MAC乘加阵列+大容量片上SRAM，硬连线控制，不可编程，功耗极低（几瓦~十几瓦）。
  • 性能瓶颈都是内存墙：算力再强，DDR带宽跟不上也是白搭。优化核心是减少访存、提高数据复用。

华为昇腾达芬奇架构

  • Scalar Unit：标量控制单元，管循环/分支/地址计算/调度。
  • Vector Unit：向量运算单元，SIMD逐元素操作（ReLU/Add/Norm）。
  • Cube Unit：矩阵乘加单元，专门干Conv/FC的MAC阵列。

MAC与算力

  • MAC = 1次乘法 + 1次加法，是NPU最小硬件单元和最小运算单位。
  • 算力换算：1024 MAC/cycle × 3核 × 1GHz × 2 = 6 TOPS（乘2是因为1MAC=2次基本运算）。

六、其他

  • TVM：模型编译器（Apache开源），输入ONNX/TensorFlow计算图，输出多硬件优化代码。和nvcc（GPU C++编译器）不同层级。
  • OpenAI Triton：Python DSL写GPU算子，底层自动生成CUDA。NVIDIA Triton Inference Server是云端推理服务框架，两者同名无关。
  • GaN器件：功率半导体，适合做电源开关（数据中心PSU、机器人关节驱动），不适合做数字MAC/AI计算。抗辐射加固CMOS FPGA已覆盖绝大多数恶劣环境数字控制需求。

四、易混淆点总结

  1. nutrient vs nutrition：nutrient 是具体的营养素（可数）；nutrition 是抽象的"营养"概念。
  2. in turn 双重含义：① 轮流（they answered in turn）；② 反过来/作为回报（A helps B, which in turn helps A）。珊瑚文章里是第二种。
  3. renown / renowned / world-renowned：renown（名望，名词）→ renowned（著名的，形容词）→ world-renowned（举世闻名的）。
  4. minuscule 拼写：不是 miniuscule，u 在 s 前面。
  5. undergraduate vs postgraduate：under（之下）= 本科；post（之后）= 研究生。
