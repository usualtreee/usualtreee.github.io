// 加载模型
rknn_context ctx;
rknn_init(&ctx, model_data, model_size, 0, NULL);
rknn_set_core_mask(ctx, RKNN_NPU_CORE_0);   // 绑 NPU Core 0

// 准备输入
cv::Mat frame = 解码出来的帧;
cv::resize(frame, frame, Size(640, 640));    // 模型输入尺寸
cv::cvtColor(frame, frame, COLOR_BGR2RGB);  // 模型要 RGB

rknn_input inputs[1];
inputs[0].buf = frame.data;
inputs[0].size = 640 * 640 * 3;
inputs[0].fmt = RKNN_TENSOR_NHWC;

// 跑推理
rknn_inputs_set(ctx, 1, inputs);
rknn_run(ctx, NULL);                        // NPU 干活，CPU 空闲

// 取输出 + 后处理
rknn_output outputs[3];
rknn_outputs_get(ctx, 3, outputs, NULL);
// 解码输出 → NMS 去重 → 坐标反算 → cv::rectangle() 画框

3. 关键知识点

  • .rknn 文件 = 网络架构 + 权重 + 量化参数，一个文件全包
  • NPU 不认 FP32，必须 INT8 量化
  • NMS（非极大值抑制）= 去掉重叠的高置信度框
  • 坐标反算 = 模型输出基于 640×640，要映射回原始分辨率

验收

  • 框跟人走，位置正确
  • 单帧推理 < 50ms

阶段 3：解码和推理分离（1~2 天）

目标

解码和推理跑在两个线程，中间用队列传递，互不阻塞。

做什么

1. 写 FrameQueue（线程安全有界阻塞队列）

1
2
3
4

// 核心三个方法
push()          // 生产者放数据，满了就阻塞等待
wait_and_pop()  // 消费者取数据，空了就阻塞等待
shutdown()      // 通知所有人下班

用 std::mutex + std::condition_variable 实现。

2. 两个线程

1
2

线程1（解码）：解完一帧 → frameQueue.push(frame)
线程2（推理）：frameQueue.wait_and_pop(frame) → 跑推理 → 画框 → 显示

3. 背压机制

1
2

解码比推理快 → 队列满 → push 阻塞 → 解码线程自动慢下来
推理比解码快 → 队列空 → pop 阻塞  → 推理线程自动等

不需要任何计时器，队列本身就能自动调节速率。

验收

  • 两个线程稳定运行不崩溃
  • 画面流畅无黑屏
  • 队列长度稳定在 2~4 帧

阶段 4：多模型并行推理（2~3 天）

目标

同一帧同时跑 3 个模型（人员/安全帽/打电话），真正并行。

做什么

1. 把推理逻辑封装成类

1
2
3
4

class rknn_lite {
    int interf(detect_result_group_t &result);  // 完整推理管线
    // 内部分步：前处理 → set_input → rknn_run → get_output → 后处理 → 画框
};

2. 模型绑定不同 NPU 核心

1
2
3

人员模型  → rknn_set_core_mask(ctx, RKNN_NPU_CORE_0);
安全帽    → rknn_set_core_mask(ctx, RKNN_NPU_CORE_1);
打电话    → rknn_set_core_mask(ctx, RKNN_NPU_CORE_2);

3. 引入线程池并行提交

1
2
3
4
5
6

// 同一帧 3 个模型同时提交
auto f1 = pool.submit([&]() { model_person->interf(result1); });
auto f2 = pool.submit([&]() { model_helmet->interf(result2); });
auto f4 = pool.submit([&]() { model_callplay->interf(result4); });

f1.get(); f2.get(); f4.get();  // 三个全跑完才继续

NPU 三个核心同时算，耗时不叠加。

验收

  • 3 个模型总耗时 ≈ 最慢那个（而不是 3 个相加）
  • NPU 利用率上升

阶段 5：RGA 硬件加速（1~2 天）

目标

用硬件替代 OpenCV 做图像预处理，释放 CPU。

做什么

1. 替换操作

1
2
3

RGA 替代：
  cv::resize()     → imresize()       // 硬件缩放
  cv::cvtColor()   → imcvtcolor()     // 硬件色彩转换（BGR → RGB）

2. DMA 零拷贝

  • 预分配 DMA32 对齐内存池
  • RGA 输入输出都在 DMA 内存里，不经过 CPU

3. Fallback 机制

1
2
3
4
5

if (!rga_done) {
    // RGA 失败，降级到 OpenCV CPU 处理
    cv::cvtColor(...);
    cv::resize(...);
}

验收

  • 前处理耗时从 ~15ms 降到 ~3ms
  • CPU 占用下降
