项目概况以及复现问题解决

第一部分：项目概况

项目是什么

这是一个基于 瑞芯微 RK3588 的多路视频 AI 智能分析系统。功能是：

4 路视频流 → 硬件解码 → NPU 并行跑 3 个 AI 模型检测 → 画面拼接 → 编码推流 → 远程观看

具体检测内容：

  • 人员检测（person）
  • 安全帽佩戴检测（helmet）
  • 打电话行为检测（callplay）

4 路画面拼接成 2×2 的合成画面，通过 RTMP/RTSP/GB28181 推流到外部平台，用 VLC 或网页实时观看。

技术栈

组件 用途
Rockchip MPP 芯片内置硬件视频解码 + 编码
RKNN (NPU) 芯片内置 6TOPS AI 推理单元
RGA 芯片内置硬件图像缩放/格式转换
OpenCV 软件图像处理、绘制检测框
FFmpeg 视频文件解封装、RTMP/RTSP 推流
eXosip2/osip2 GB28181 国标 SIP 协议栈
C++17 / CMake 构建系统

第三部分：踩坑记录与解决方案

踩坑 1：NPU 推理段错误（rknn_server 未启动）

现象 

1
2

$ ./build/myDemo 1 ./assets/videos/1.mp4
段错误 (核心已转储)

排查 

1
2

ls -la /dev/rknpu*        # 不存在
dmesg | grep -i npu       # 内核驱动已初始化，但设备节点没创建

根因 

1

程序 → librknnrt.so → rknn_server（常驻进程）→ /dev/rknpu → NPU 硬件

rknn_server 是用户空间的后台服务进程，没有它，后续任何 NPU 调用都是空指针 → 段错误。

解决 

1
2

sudo cp third_party/rknn/bin/rknn_server /usr/bin/
sudo rknn_server &

踩坑 2：MPP 硬件解码段错误（库版本不匹配）

现象

GDB 定位到 hal_h264e_vepu580_init → mpp_init → MppEncoder::Init 崩溃。

排查 

1
2
3

ldd ./build/myDemo | grep mpp
# → librockchip_mpp.so.1 => /usr/lib/aarch64-linux-gnu/librockchip_mpp.so.1
# 但 third_party 里只有 librockchip_mpp.so，没有 .so.1！

根因

程序链接 librockchip_mpp.so.1，但 third_party 里没有版本后缀，动态链接器跳过去找系统旧版 → API 不兼容 → 段错误。

解决 

1
2
3

cd third_party/mpp/lib
sudo ln -sf librockchip_mpp.so librockchip_mpp.so.1
sudo ln -sf librockchip_mpp.so librockchip_mpp.so.0

踩坑 3：权限问题

普通用户无法与 root 启动的 rknn_server 通信 → sudo su -c 解决。

踩坑 4：RGA DMA 物理地址为空（不致命）

现象 

1
2

RgaBlit(1513) RGA_BLIT fail: Invalid argument
buffer[handle,fd,va,pa] = [0, 0, 0x..., 0]   ← pa 全是 0

根因

RGA 库版本跟板子内核驱动 ioctl 接口不完全匹配，DMA buffer 分配失败。

解决 

1
2
3
4
5

# 替换为系统自带的匹配 RGA 库
mv third_party/rga/lib/librga.so third_party/rga/lib/librga.so.bak
sudo cp /usr/lib/aarch64-linux-gnu/librga.so.2.1.0 third_party/rga/lib/librga.so
# 放权 DMA heap 设备
sudo chmod 666 /dev/dma_heap/*

当前状态：RGA 偶尔失败时回退到 CPU 处理，不影响数据通路，视频正常推流。

踩坑总结

# 问题 症状 根因 解决方案
1 rknn_server 未启动 段错误 NPU 用户态服务缺失 拷贝 + 后台启动
2 MPP 库版本不匹配 段错误 .so 缺版本后缀 ln -sf 补软链接
3 权限问题 偶尔段错误 普通用户无法访问 rknn_server sudo 运行
4 RGA DMA 物理地址为空 Invalid argument RGA 库/驱动 ioctl 不匹配 + CMA 边界 换系统 RGA 库 + 放权 dma_heap

第四部分：永久性环境修改

# 改动 路径 类型
1 安装软件包 系统 apt 新增
2 部署 rknn_server /usr/bin/rknn_server 新增文件
3 MPP 版本软链接 third_party/mpp/lib/*.so.1/.so.0 新增链接
4 替换 RGA 库 third_party/rga/lib/librga.so 覆盖文件（原文件备份）
5 放权 DMA heap /dev/dma_heap/* 权限 666 临时（重启失效）

所有改动均在项目 third_party/ 或系统 /usr/bin/ 下，项目源码一字未改。
