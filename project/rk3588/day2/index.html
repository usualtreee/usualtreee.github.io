学习指南

第一阶段：你需要什么基础

在打开代码之前，先把这 5 件事搞懂（每一件都不需要精通，但要知道它在干什么）：

1. C++ 多线程基础（2 天）

你需要理解 3 个概念：

  • std::thread — 怎么开一个线程
  • std::mutex + std::lock_guard — 多个线程抢同一个变量怎么办
  • std::condition_variable — 生产者通知消费者"有货了"，消费者等待

这个项目的核心代码就是用这三样东西写的，FrameQueue、线程池、推理线程全是。

2. FFmpeg 基本概念（1 天）

不需要会写代码，但要明白 4 个术语：

  • 解封装（Demux） — 把 RTSP/MP4 文件拆成视频流 + 音频流
  • AVPacket — 一包压缩后的视频数据（H.264 的码流片）
  • AVFormatContext — 代表"一个视频文件或一个流地址"
  • AVBSFContext — 码流格式转换器（比如把封装格式转成纯 AnnexB 格式）

3. 视频编解码常识（半天）

  • H.264/H.265 是什么
  • 解码 = 压缩数据 → 原始像素（YUV/RGB）
  • 编码 = 原始像素 → 压缩数据
  • 关键帧（I 帧）vs 非关键帧（P/B 帧）：解码必须从 I 帧开始

4. 边缘计算 NPU 概念（半天）

  • CPU/GPU/NPU 的区别：NPU 是专门跑神经网络的硬件，不占 CPU
  • 模型推理 = 输入图片 → 神经网络计算 → 输出检测框坐标 + 置信度
  • 量化：把浮点模型（Float32）转成定点模型（INT8），牺牲一点精度换速度，RK3588 NPU 要求 INT8 格式的 .rknn 文件

5. RK3588 硬件模块认知（半天）

  • MPP — 瑞芯微的媒体处理框架，硬件编解码（比软解快 10 倍）
  • RGA — 瑞芯微的图像加速模块，硬件做缩放/旋转/色彩转换（比 OpenCV 快 3-5 倍）
  • RKNN — 瑞芯微的 NPU 推理框架，加载 .rknn 模型文件跑推理

第四阶段：按依赖关系读源码（从底到顶）

不要从 main.cpp 开始看！按下面这个顺序，每一步都只需要理解上一个步骤的内容：

第 1 步：线程安全队列 frame_queue.h（20 分钟）

这是整个项目最底层的轮子。只看 3 个方法：

  • push() — 生产者放数据，队列满了就阻塞等待
  • wait_and_pop() — 消费者取数据，队列空了就阻塞等待
  • shutdown() — 通知所有人下班

验收：能画出 push 和 wait_and_pop 的互斥流程图。

第 2 步：线程池 ThreadPool.hpp（30 分钟）

只看 submit() 和 worker() 两个方法：

  • submit() — 把任务打包 + 放进队列 + 通知工作线程
  • worker() — 死循环等任务 → 取任务 → 执行 → 如果 2 秒没活就退出

验收：能说清楚一个任务从 submit 到执行完的完整路径。

第 3 步：性能监控 performance_monitor.hpp（5 分钟）

就是 start/end 记录耗时，最后 printStats 打平均值。纯工具类。

第 4 步：MPP 硬解码 mpp_decoder.h（40 分钟）

这是 RGB→YUV 领域的核心。理解：

  • Init() — 初始化 MPP 解码器，指定 H.264/H.265 类型
  • Decode() — 喂一包压缩数据（AVPacket），MPP 内部解码后回调通知
  • 回调机制：解码完成 → 调用外部注册的函数，把帧数据传出去

验收：能说清楚"一个 AVPacket 怎么变成一个解码后的帧"。

第 5 步：拉流封装 stream_loader.h（40 分钟）

把 FFmpeg 拉流和 MPP 解码串起来：

  • open() — FFmpeg 打开 RTSP/MP4，找到视频流索引，初始化 MPP 解码器
  • read_frame() — FFmpeg av_read_frame() 读一包 → 塞进 MPP 解码器
  • operator()() — 死循环 read_frame，实现持续解码

验收：能画出一路流的完整数据管道。

第 6 步：RKNN 推理封装 rknnPool.hpp（1 小时）

这是核心中的核心：

  • 构造函数：加载 .rknn 模型文件 → rknn_init() 初始化 → rknn_set_core_mask() 绑 NPU 核心 → 查询输入输出属性
  • interf()：完整推理管线
    1. RGA 硬件 Resize + BGR→RGB（带 OpenCV fallback）
    2. rknn_inputs_set() 喂数据
    3. rknn_run() 跑推理
    4. rknn_outputs_get() 取结果
    5. 后处理 NMS + 坐标反算
    6. 在图上画框

验收：能说清楚 interf() 里的 6 个步骤和每一步用了什么硬件。

第 7 步：多模型融合 detection_fusion_manager.h（30 分钟）

  • fuseDetections() — 把 4 个模型的检测结果融合成 1 份，IoU 去重 + IoM 处理嵌套
  • drawFusedDetections() — 在图上画出融合后的框

第 8 步：推流管理 streaming_manager.h（30 分钟）

  • 生产者-消费者：合成线程往里塞帧，推流线程往外掏 → MPP 编码 → FFmpeg 封包发送
  • 支持 RTMP 和 RTSP 两种输出

第 9 步：入口 main.cpp（30 分钟）

现在才看 main.cpp，你会觉得一目了然：

  1. 解析命令行参数（几路流、流地址）
  2. 初始化模型路径 + 推流配置
  3. 核心循环：给每路流创建解码线程 + 4 个模型推理线程 → 提交到线程池
  4. combineImage 线程：4 路画面合成 2×2 → 推流
