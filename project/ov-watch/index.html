OV-Watch-项目概况

第一部分：项目概况

项目是什么

基于 OV-Watch 开源项目，在 STM32F411CEU6（ARM Cortex-M4，100MHz）平台上完成智能手表嵌入式软件系统的开发与调试。功能如下：

硬件初始化 → 多传感器数据采集 → FreeRTOS 多任务调度 → LVGL 渲染 UI → 触摸/按键交互

具体功能：

  • 心率监测（PPG 信号 + 峰值检测算法）
  • 环境感知（温湿度 + 气压高度 + 电子罗盘）
  • 运动追踪（IMU 计步 + 抬腕唤醒）
  • 蓝牙 OTA 无线升级（独立 BootLoader + Ymodem 协议）
  • 工具应用（计算器、秒表/倒计时、日历）
  • 休闲游戏（2048、记忆翻牌）

支持 PC 仿真调试（SDL2 + CMake），无需真实硬件即可开发 UI。

核心数据通路说明

  1. 按键/触摸输入：KeyTask 扫描按键 → 消息队列 → PageManager 页面栈（深度 6 层）→ 决定当前显示的 LVGL 页面
  2. 传感器采集：7 类传感器通过 I2C/SPI 总线采集 → SensorDataTask 定时轮询 → 通过发布-订阅（pubsub）通知 UI 刷新
  3. LVGL 渲染：ScrRenewTask 周期性调用 lv_task_handler() → LVGL 计算脏区域 → 刷新 ST7789 LCD
  4. 低功耗：空闲计时器（100ms 周期）→ 无操作超时 → IdleEnterTask 触发 STOP 模式 → RTC 中断唤醒
  5. 蓝牙 OTA：手机通过 SPP 发送 .bin 固件 → BootLoader Ymodem 协议接收 → Flash 校验写入 → 跳转新 APP

第二部分：关键设计要点

1. 硬件抽象层（HWDataAccess）

User/Func/Inc/HWDataAccess.h 是整个项目移植性的核心。通过顶层宏 HW_USE_HARDWARE 一键切换编译模式：

1
2

#define HW_USE_HARDWARE 1   // Keil 工程：访问真实硬件
#define HW_USE_HARDWARE 0   // VS Code 仿真：全部函数返回模拟数据

所有硬件接口通过结构体函数指针封装（HW_InterfaceTypeDef），UI 层完全不直接操作硬件寄存器。切换仿真模式时，所有 HW_USE_* 子宏被跳过，底层 BSP 代码不被编译，UI 代码零改动即可在 PC 上运行。

2. 页面栈导航

User/Func/Inc/PageManager.h 使用栈结构管理页面跳转历史，深度 6 层：

API 功能
Page_Load(newPage) 入栈新页面，加载并显示
Page_Back() 出栈当前页，回退到上一页
Page_Back_Bottom() 清栈，返回首页

每个页面被抽象为 Page_t 结构体（init / deinit / page_obj 三个成员），新页面只需注册此结构体即可纳入导航体系。

3. 发布-订阅模式

User/Func/Src/pubsub.c 用单向链表实现了轻量级发布-订阅机制，用于任务间解耦通信：

  • Publisher_subscribe() — 注册订阅者回调函数
  • Publisher_publish() — 遍历链表通知所有订阅者
  • Publisher_unsubscribe() — 移除订阅者

典型用例：传感器数据更新后 → Publisher_publish() → UI 页面收到通知 → 刷新显示值。

4. 双栈计算器

StrCalculate.c 实现了一个经典的双栈表达式求值器：

1
2
3
4
5
6

输入 "1+2*6/3" → 遍历字符
  ├─ 数字 → 压入操作数栈（支持小数：小数点后位逐位 ÷10 累加）
  ├─ 运算符 → 与符号栈顶比较优先级
  │    ├─ 当前优先级 ≤ 栈顶 → 弹出两数 + 一符号 → 计算 → 结果入栈
  │    └─ 当前优先级 > 栈顶 → 直接入栈
  └─ 遍历结束 → 依次弹出栈内剩余运算 → 得到最终结果

5. 三级电源管理

模式 状态 功耗 唤醒方式
运行 MCU 正常运行，所有外设供电 70-80mA —
睡眠 MCU STOP 模式，MPU6050 保持计步，蓝牙断电 ~1mA RTC 定时中断 + 姿态检测
关机 TPS63020 断电，仅 VBAT 维持 RTC 微安级 长按 KEY1

关键设计：抬腕唤醒并非使用 MPU6050 运动中断（抖动阈值过高，不灵敏），而是RTC 定时唤醒 + 软件读取当前姿态，灵敏度大幅提升。

6. 蓝牙 OTA 固件升级 

1
2
3
4
5
6

APP 运行中 → 收到 OTA 指令 → 跳转 BootLoader（0x0000C000）
  → 蓝牙 SPP 连接手机
  → Ymodem 协议接收 .bin 文件包
  → Flash 擦写（APP 区，0x0000C000 起）
  → CRC 校验
  → 跳转新 APP 入口

BootLoader 和 APP 通过 FLASH 分区隔离（BootLoader 48KB，APP 从 0x0000C000 开始），中断向量表重映射由 SCB->VTOR = 0x0000C000U 完成（main.c:75）。

7. PC 仿真器

lv_sim_vscode_win/ 提供了完整的 Windows 仿真环境：

  • 构建系统：CMake + SDL2，VS Code 一键构建
  • 输入模拟：鼠标（触屏）、键盘（编码器）、滚轮
  • 显示模拟：SDL2 窗口绘制 LVGL 渲染缓冲区
  • 硬件隔离：HW_USE_HARDWARE → 0，所有传感器函数返回模拟数据

含义：UI 开发 → 仿真器验证 → 拷贝 Func/ 和 GUI_App/ 到 Keil 工程 → 真机运行，零代码改动。

第三部分：踩坑记录与解决方案

踩坑 1：程序启动后立即卡死（中断向量表未重映射）

现象

不带 BootLoader 编译 APP 下载后正常运行；带 BootLoader 后，APP 启动即卡死在 HAL_Init() 中。

排查 

1

// 对比 BootLoader 和无 Boot 版本代码差异

根因

STM32F4 默认中断向量表基址在 0x08000000，但 BootLoader 占用前 48KB，APP 的向量表在 0x0800C000。APP 启动后发生中断时，CPU 仍去 0x08000000 取向量 → 取到已经跳走的 BootLoader 中断处理 → 行为不可预期。

解决

APP 的 main() 第一行必须设置中断向量表偏移：

1

SCB->VTOR = 0x0000C000U;

踩坑 2：睡眠模式功耗异常偏高

现象

进入 STOP 模式后实测功耗远高于预期（几十 mA 而非 ~1mA）。

排查

逐个外设 DeInit，定位到 USART1（蓝牙串口）。

根因

进入睡眠前只是关了蓝牙使能，未将 USART1 的 TX/RX GPIO 重新配置为模拟输入（高阻态）。引脚浮空产生漏电流。

解决

进入 STOP 前 DeInit USART1，将 PA9/PA10 设为模拟输入模式。出 STOP 后重新 Init。

踩坑 3：MPU6050 DMP 初始化后功耗高

现象

MPU6050 Init + DMP 启动后，整体功耗明显偏高。

根因

DMP 库默认配置使 MPU6050 一直处于全速采样模式，在睡眠模式下仍然如此。

解决

修改 MPU6050 初始化流程：仅在需要计步和姿态检测时使能 DMP，持续空闲后降低采样率。具体见 BSP/MPU6050/ 中的配置改动。

踩坑 4：页面栈弹出返回时偶发 UI 错误

现象

从子页面返回时，偶尔显示白屏或上一次页面残留。

排查

检查 PageManager.c 中的栈弹出逻辑。

根因

PageStack 中存储的是页面对象指针，但 LVGL 的 lv_obj_t 在 lv_scr_load_anim() 切换后可能被回收。栈中存储的指针变成了悬空指针。

解决

改为在栈中存储页面初始化和去初始化的函数地址（Page_t 结构体），返回时重新调用 init() 重建页面，而非直接复用旧对象指针。

踩坑 5：LVGL 字体渲染导致编译固件过大

现象

勾选所有 UI 字体后，Keil 编译出的固件超出 FLASH 容量。

根因

醋园体中文字库每个字号都包含完整 GB2312 字集，多个字号叠加后远超 512KB。

解决

仅保留 UI 实际使用的字号（16/18/20/24/28/32/40/48），关闭不需要的字号宏。图标字库同理。

踩坑 6：仿真器鼠标坐标映射偏移

现象

VS Code 仿真器中点击位置与实际 UI 位置有偏移。

根因

SDL2 窗口大小与 LVGL 显示分辨率不一致，触屏驱动没有做坐标缩放。

解决

在仿真器的输入驱动中添加 zoom 系数转换：lv_x = SDL_x * LVGL_HOR_RES / SDL_WINDOW_W。

踩坑 7：EEPROM 写入后立即断电导致数据丢失

现象

修改设置 → 关机 → 再开机，设置恢复默认值。

根因

EEPROM 写周期 ~5ms，如果在写完成前掉电，该字节写入不完整。Datasave 没有引入写延迟和校验。

解决

在 DataSaveTask 的消息队列处理中，写入后延时 10ms 确保 I2C 传输完成；读取时增加 magic number 校验，校验失败回退默认值。

踩坑总结

# 问题 症状 根因 解决方案
1 中断向量表未偏移 BootLoader 后 APP 卡死 VTOR 仍指向 0x08000000 SCB->VTOR = 0x0000C000U
2 睡眠功耗高 STOP 模式几十 mA UART GPIO 未设高阻态 DeInit USART1 + GPIO 设模拟输入
3 MPU6050 功耗高 DMP 初始化后整体功耗高 全速采样未降频 空闲时降低采样率
4 页面返回 UI 错乱 返回时白屏或残留 lv_obj 悬空指针 返回时重调 init() 而非复用对象
5 固件超 FLASH Keil 编译不过 过多字库 仅保留实际使用的字号
6 仿真器点击偏移 点击不准 窗口与 LVGL 分辨率不匹配 zoom 系数坐标转换
7 EEPROM 数据丢失 断电后设置恢复 写周期内掉电 写入后延时 + magic number 校验
