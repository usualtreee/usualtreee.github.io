  By not providing "FindSDL2.cmake" in CMAKE_MODULE_PATH this project has
  asked CMake to find a package configuration file provided by "SDL2", but
  CMake did not find one.

  Could not find a package configuration file provided by "SDL2" with any of
  the following names:

    SDL2Config.cmake

排查

  1. 项目目录下确实有 SDL2.dll（运行时库），但缺少编译所需的头文件和 CMake 配置文件
  2. MinGW 的 lib/ 下有 libSDL2.a（链接库），但 include/ 下没有 SDL2/ 目录（头文件）
  3. 系统没有 sdl2-config.cmake，find_package 无法定位 SDL2

总结：项目只带了 SDL2.dll（运行时），缺少 SDL2 开发包（头文件 + CMake 配置）。

解决

Step 1：下载 SDL2 开发包

curl -L -o /tmp/SDL2-devel-mingw.zip \
  "https://github.com/libsdl-org/SDL/releases/download/release-2.30.12/SDL2-devel-2.30.12-mingw.zip"

# 拷贝头文件
cp -r /tmp/sdl2-devel/SDL2-2.30.12/x86_64-w64-mingw32/include/SDL2 \
  "F:/application/w64devkit/include/"

# 拷贝库文件
cp /tmp/sdl2-devel/SDL2-2.30.12/x86_64-w64-mingw32/lib/libSDL2.a \
  "F:/application/w64devkit/lib/"
cp /tmp/sdl2-devel/SDL2-2.30.12/x86_64-w64-mingw32/lib/libSDL2main.a \
  "F:/application/w64devkit/lib/"
cp /tmp/sdl2-devel/SDL2-2.30.12/x86_64-w64-mingw32/lib/libSDL2.dll.a \
  "F:/application/w64devkit/lib/"

cmake .. -G "MinGW Makefiles" \
  -DSDL2_DIR="/tmp/sdl2-devel/SDL2-2.30.12/cmake"

根因

CMake 的 find_package(SDL2 REQUIRED) 需要 sdl2-config.cmake，这个文件只存在于 SDL2 开发包中。

踩坑 2：仿真器鼠标交互卡顿、手势难以触发

现象

仿真器窗口弹出来后：

  • 鼠标拖拽页面很难滑动
  • 从主页右滑进入菜单后，左滑回不去
  • 键盘方向键也不管用

排查

1. 键盘方向键为什么不灵？

看了 lv_drivers/sdl/sdl.c 的键盘处理代码（第 572-600 行），方向键确实被正确映射为 LV_KEY_RIGHT/LV_KEY_LEFT。但这些键是给 LVGL 控件组导航用的——手表 UI 用的是触摸手势（LV_EVENT_GESTURE），根本没用控件组。所以方向键无效。

真正的返回键是 Backspace（退格键）：

  • sdl.c:336-340 — 检测到 data->key == 8（LV_KEY_BACKSPACE）→ 发布 "SDL_Key" 消息
  • ui.c:17-21 — 订阅该消息 → 调用 Page_Back()

所以键盘上 Backspace = 返回，不是 ESC，不是方向键。

2. 鼠标拖拽手势为什么难触发？

LVGL 手势检测（lv_hal_indev.c）要求同时满足两个条件：

参数 默认值 含义
gesture_limit 50px 最少拖拽距离
gesture_min_velocity 3 最少拖拽速度

问题一：仿真器窗口 SDL_ZOOM = 1，只有 240×280 像素——太小了，50px 占比太大。

问题二：鼠标拖本来就很慢，速度达不到阈值 3，所以慢拖永远不触发手势。

解决

修改 lv_drv_conf.h（第 102 行）：窗口放大一倍

1
2
3
4

// 改前
#define SDL_ZOOM 1
// 改后
#define SDL_ZOOM 2   // 窗口变成 480x560，鼠标操作更方便

修改 main.c（第 148-150 行后）：降低手势门槛

1
2
3

// 新增
indev_drv_1.gesture_limit = 30;          // 默认 50px → 30px
indev_drv_1.gesture_min_velocity = 0;    // 默认 3 → 0（关掉速度检查）

根因

LVGL 手势检测的两个默认值是为手指触摸设计的（手指快、拖距短）。鼠标操作特性完全不同（精度高但速度慢），直接用默认值会导致手势识别率极低。

永久性改动

改动 文件 说明
SDL_ZOOM: 1 → 2 lv_drv_conf.h:102 窗口放大一倍
新增 gesture_limit = 30 main.c:148-150 降低最小拖拽距离
新增 gesture_min_velocity = 0 main.c:148-150 取消速度限制

仿真器操作方式总结

操作 对应真机 按键/动作
鼠标单击 手指点一下 点按钮、点菜单项
鼠标按住拖拽 手指滑动 主页右滑→菜单、左滑/Backspace→返回
Backspace 键 物理返回键 任意页面返回上一页

踩坑 3：左滑手势无效（部分页面只响应右滑）

现象

  • 主页右滑能进菜单 ✓
  • 菜单右滑能返回 ✓
  • 菜单左滑无反应 ✗（本应也返回）
  • 设置页、游戏选择页、日期设置页同样问题

排查

搜索所有页面的手势处理代码：

1

grep -n "LV_DIR_RIGHT\|LV_DIR_LEFT" user_test/GUI_App/Screens/Src/*.c

结果：所有页面只判断了 LV_DIR_RIGHT，没有判断 LV_DIR_LEFT。

以 ui_MenuPage.c:98-104 为例：

1
2
3
4
5
6
7

if(event_code == LV_EVENT_GESTURE)
{
    if(lv_indev_get_gesture_dir(lv_indev_get_act()) == LV_DIR_RIGHT)
    {
        Page_Back();  // 只右滑能触发，左滑被忽略
    }
}

解决

修改 4 个页面（MenuPage、SetPage、GameSelectPage、DateTimeSetPage），把方向判断从单一右滑改为左右都响应：

1
2
3
4
5
6

// 改前
if(lv_indev_get_gesture_dir(lv_indev_get_act()) == LV_DIR_RIGHT)

// 改后
if(lv_indev_get_gesture_dir(lv_indev_get_act()) == LV_DIR_RIGHT ||
   lv_indev_get_gesture_dir(lv_indev_get_act()) == LV_DIR_LEFT)

根因

真机手表戴在手腕上，拇指自然只做右滑操作，作者只写了右滑。仿真器上鼠标拖拽方向不固定，用户习惯左滑返回。

踩坑 4：计算器、心率等页面完全没有手势功能

现象

从菜单进入计算器、心率、指南针、日历、计时器等页面后，左右滑都没反应，只能用 Backspace 返回。

排查

搜索发现这些页面根本没有 LV_EVENT_GESTURE 处理代码：

1
2
3

有手势的页面：HomePage, MenuPage, SetPage, GameSelectPage, Game2048Page, DateTimeSetPage
无手势的页面：ComputerPage, HRPage, SPO2Page, CompassPage, EnvPage, CalendarPage,
             TimerPage, ChargPage, AboutPage, NFCCardPage, OffTimePage, GameMemPage

12 个页面全都没有手势处理——每个加 3 行代码太蠢了。

解决

在 PageManager 里加通用手势回调，所有页面自动获得左滑返回能力。

1. PageManager.h — Page_t 结构体加字段：

1
2
3
4
5
6

typedef struct {
    void (*init)(void);
    void (*deinit)(void);
    lv_obj_t **page_obj;
    bool no_auto_swipe_back;   // true = 不自动加左滑返回
} Page_t;

C 语言零初始化：未指定的结构体字段默认为 0（false），所以所有已有的 Page_t Page_XXX = {...} 声明不用改，自动获得左滑返回。

2. PageManager.c — 通用回调 + Page_Load 里自动挂载：

 3

// 通用手势回调：左滑 → 返回
static void auto_swipe_back_handler(lv_event_t * e)
{
    if(event_code == LV_EVENT_GESTURE)
    {
        if(lv_indev_get_gesture_dir(lv_indev_get_act()) == LV_DIR_LEFT)
            Page_Back();
    }
}

// Page_Load 末尾加一行
if (!newPage->no_auto_swipe_back)
    lv_obj_add_event_cb(*newPage->page_obj, auto_swipe_back_handler, LV_EVENT_ALL, NULL);

3. 例外页面 — 2048 和 HomePage 标为 true：

1
2
3
4
5

// ui_Game2048Page.c — 左滑是游戏操作，不能抢占
Page_t Page_Game_2048 = {..., true};

// ui_HomePage.c — 已经是底部，没有可返回的
Page_t Page_Home = {..., true};

根因

原项目只有少数页面加了手势，大量页面只靠物理按键返回。仿真器物理按键不方便，需要手势补充。

永久性改动

改动 文件 说明
Page_t 加 no_auto_swipe_back 字段 PageManager.h 标记不自动挂手势的页面
通用 auto_swipe_back_handler PageManager.c 左滑自动返回
Page_Load 里自动挂载回调 PageManager.c 所有页面自动获得左滑返回
LV_DIR_LEFT 支持 4 个页面 .c 已有手势的页面补上左滑
2048 / HomePage 标 true 2 个页面 .c 排除例外
