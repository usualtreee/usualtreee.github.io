OV-Watch — 学习总结 Day 1

一、项目是什么

OV-Watch：基于 STM32F411CEU6（ARM Cortex-M4）的智能手表，FreeRTOS + LVGL v8.2，19 个 UI 界面，14 项功能（心率、计步、指南针、蓝牙 OTA、计算器、2048 等）。

二、仿真器搭建

做了什么

  1. 装 CMake + MinGW（w64devkit）
  2. 补装 SDL2 开发包（项目只带了 SDL2.dll 运行时，缺头文件和 CMake 配置）
  3. cmake .. -G "MinGW Makefiles" -DSDL2_DIR=... 配置
  4. make -j4 编译
  5. ../bin/main.exe 运行——弹出 480×560 手表 UI 窗口

踩坑

  • 坑 1：CMake 找不到 SDL2 → 下载 SDL2-devel-mingw.zip，拷头文件到 MinGW include，拷 .a 到 MinGW lib
  • 坑 2：仿真器窗口太小 + 手势难以触发 → lv_drv_conf.h 中 SDL_ZOOM 从 1 改为 2；main.c 中鼠标手势 gesture_limit 降为 30，gesture_min_velocity 降为 0
  • 坑 3：左滑无反应 → 四个页面（MenuPage/SetPage/GameSelectPage/DateTimeSetPage）的 LV_DIR_RIGHT 判断补上 LV_DIR_LEFT
  • 坑 4：计算器、心率等 12 个页面完全没有手势 → PageManager 加通用左滑返回回调，2048 和 HomePage 标 true 排除

仿真器操作总结

操作 对应 按键/动作
鼠标单击 手指点触 点按钮
鼠标按住拖拽 手指滑动 切页/返回
Backspace 键 物理返回键 任意页面返回

键盘方向键不生效——手表 UI 用的是触摸手势，不是 LVGL 控件组导航。

关键认知

  • 仿真器代码和 Keil 代码不共享：在仿真器 user_test/ 改完 UI，手动复制到 Software/OV_Watch/User/ 覆盖，切换宏 HW_USE_HARDWARE 即可
  • LVGL 是源码编译：不是装库，是把 lvgl/ 整个目录的 .c 文件跟自己的代码一起编进固件
  • FreeRTOS 同理：源码在 Middlewares/Third_Party/FreeRTOS/，一套 FreeRTOSConfig.h 决定一切

Drivers vs BSP

  • Drivers（ST 官方）：通用 HAL 层，HAL_I2C_Read() 怎么发 I2C 数据
  • BSP（作者写）：知道每颗外设芯片的寄存器地址，MPU6050_GetSteps() 发什么数据读到步数

仿真器（PC）：
  main.c → lv_init() → hal_init() → sdl_init()
         → lv_timer_handler() 每 5ms 调一次
         → SDL2 把 LVGL 像素画到电脑窗口
         → SDL2 把鼠标事件转成 LVGL 触摸事件
         → HW_USE_HARDWARE = 0，传感器全返回假数据

真机（STM32）：
  main.c → FreeRTOS 调度
         → LvHandlerTask 每 1ms 调 lv_timer_handler()
         → LVGL 像素直接写到 ST7789 LCD
         → CST816 触摸事件
         → HW_USE_HARDWARE = 1，传感器真实数据

六、BootLoader + OTA 核心概念

  • FLASH 分区：BootLoader（0x08000000，48KB）+ APP（0x0800C000，464KB）
  • APP 第一行 SCB->VTOR = 0x0000C000U —— 重映射中断向量表
  • OTA 流程：手机蓝牙 SPP → Ymodem 协议分包传输 → BootLoader 校验 → 写入 FLASH → 重启运行新 APP
  • HEX vs BIN：.hex 带地址信息（ST-Link 用），.bin 纯机器码（OTA 用）
  • NoBoot.hex = BootLoader.hex + APP.bin 合并（一次烧完）

七、CMSIS-RTOS v2

ARM 公司定的 RTOS 标准 API 接口。项目代码写 osThreadNew()、osDelay()、osMessageQueuePut()，底层翻译成 FreeRTOS 的 xTaskCreate()、vTaskDelay()、xQueueSend()。它不是 ST 要求的，是 ARM 推的"换 RTOS 不用改代码"的标准。
