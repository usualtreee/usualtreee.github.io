OV-Watch — 从 0 构建项目

从 0 构建项目

假设你有一块 STM32F411CEU6 + 一模一样的硬件（传感器、屏幕、蓝牙模块等），手上只有一个空白文件夹。

第一步：CubeMX 生成框架

用 STM32CubeMX 图形化配置，点 “GENERATE CODE”，得到骨架工程。

1.1 基础设置

项 值
MCU 型号 STM32F411CEU6
时钟源 HSI → PLL → 100MHz（PLLM=8, PLLN=100, PLLP=2）
调试接口 Serial Wire（SWD）

1.2 配置外设引脚

对照硬件原理图（Hardware/SCH_Core_2024-05-26.pdf），在 CubeMX 里把每个外设的引脚配好：

外设 类型 引脚 CubeMX 配置
LCD SPI1 PA5(SCK)/PA7(MOSI)/PB6(CS)/PB7(DC)/PA15(RST) SPI1 Full-Duplex Master
触摸屏 I2C1 PB8(SCL)/PB9(SDA) I2C1
IMU I2C1 PB8/PB9 同上（挂同一根总线）
温湿度 I2C1 PB8/PB9 同上
罗盘 I2C1 PB8/PB9 同上
气压 I2C1 PB8/PB9 同上
蓝牙 USART1 PA9(TX)/PA10(RX) USART1 Async
RTC RTC — 启用 LSE 外部 32.768kHz 晶振
LCD 背光 TIM3_CH3 PB0 PWM Generation
按键 KEY1 GPIO Input PA0 上拉输入
按键 KEY2 GPIO Input PA1 上拉输入
蓝牙使能 GPIO Output PB1 推挽输出
电源使能 GPIO Output PA4 推挽输出
看门狗 IWDG — 启用独立看门狗

1.3 配置中间件

项 值
RTOS FreeRTOS，CMSIS-RTOS v2 封装
Tick Rate 1000Hz（1ms）
Heap Size 16KB
Stack Size 128 words × 任务

第三步：写 BSP 驱动

按照依赖顺序逐个写。

3.1 系统延时

SYSTEM/delay.c：用 SysTick 写个微秒级延时，所有驱动初始化时都依赖它。

    // 调 HAL_Delay 或循环

3.2 I2C 封装

BSP/IIC/：其他传感器全部挂在 I2C1 上，先封装好统一的读写。

1
2

uint8_t I2C_ReadByte(uint8_t dev_addr, uint8_t reg_addr);
uint8_t I2C_WriteByte(uint8_t dev_addr, uint8_t reg_addr, uint8_t data);

3.3 LCD 屏幕

BSP/LCD/：ST7789 SPI 驱动。屏幕亮了才有调试手段。

1
2
3

void LCD_Init(void);
void LCD_Fill(uint16_t x1, uint16_t y1, uint16_t x2, uint16_t y2, uint16_t color);
void LCD_ShowString(uint16_t x, uint16_t y, uint8_t *str, ...);

初始化顺序（查 ST7789 数据手册）：硬件复位 → 软件复位 → 睡眠退出 → 设置像素格式 → 设置行/列地址 → 开显示。

3.4 按键

BSP/KEY/：GPIO 读取 + 去抖。

1

uint8_t KEY_Scan(void);  // 返回 0=没按, 1=KEY1, 2=KEY2

3.5 各传感器

每个传感器的套路：读芯片数据手册 → 知道寄存器地址 → I2C_ReadByte/WriteByte 读写 → 封装接口。

传感器 文件夹 I2C 地址 核心函数
MPU6050 MPU6050/ 0x68 MPU6050_Init()、MPU6050_GetSteps()、抬腕检测
EM7028 EM7028/ — EM7028_Init()、EM7028_ReadPPG()
AHT21 AHT21/ 0x38 AHT21_Init()、AHT21_GetHumiTemp()
LSM303DLH LSM303DLH/ 0x1E LSM303DLH_Init()、读方向角
SPL06-001 SPL06_001/ 0x76 SPL06_001_Init()、读气压/海拔
CST816 TOUCH/ — CST816_Init()、读触摸坐标

3.6 蓝牙

BSP/KT6328/：初始化 UART → KT6368A 初始化 → 蓝牙 SPP 配对。

3.7 电源管理

BSP/POWER/：控制 TPS63020 使能、电池电量 ADC 读取。

1
2
3

void Power_Init(void);
void Power_Shutdown(void);     // 关 TPS63020，仅 RTC 存活
uint8_t BatCalculate(void);   // ADC 读电池电压 → 算百分比

3.8 EEPROM 存储

BSP/BL24C02/DataSave.c：I2C EEPROM 读写，用户设置持久化。

3.9 看门狗

BSP/OWDG/：独立看门狗，喂狗失败则 MCU 自动复位。

13 个任务：
  HardwareInitTask    ← 开机跑一次，初始化所有外设 → 挂起
  WDOGFeedTask        ← 每 100ms 喂狗
  KeyTask             ← 每 10ms 扫描按键 → 发消息
  ScrRenewTask        ← 收消息 → 切换页面
  SensorDataTask      ← 定时读温湿度/气压/罗盘
  HRDataTask          ← 读心率 + 跑峰值检测算法
  MPUCheckTask        ← 抬腕检测 → 唤醒
  LvHandlerTask       ← 每 1ms 调 lv_task_handler()
  ChargPageEnterTask  ← 检测充电
  DataSaveTask        ← 存 EEPROM
  MessageSendTask     ← 蓝牙发数据
  IdleEnterTask       ← 超时无操作 → STOP 睡眠
  StopEnterTask       ← 长按 KEY1 → 关机

6 个消息队列：
  Key_MessageQueue、Idle_MessageQueue、Stop_MessageQueue、
  IdleBreak_MessageQueue、HomeUpdata_MessageQueue、DataSave_MessageQueue

int main(void) {
    HAL_Init();
    SystemClock_Config();
    // 初始化最小外设（LCD、按键、蓝牙）
    LCD_Init();
    KT6328_Enable();
    Key_Port_Init();
    
    if (按住 KEY1 开机) {
        FLASH_If_Init();
        Main_Menu();         // ST 官方代码，等 Ymodem 传输
    } else if (FLASH 里有 APP) {
        跳转 APP;           // VTOR = 0x0800C000，跳转
    } else {
        LCD_ShowString("No App!");
        while(1);
    }
}

整个工作量评估

步骤 耗时（估计） 谁做
CubeMX 生成框架 半天 自己配
加入 LVGL + FreeRTOS 半天 复制粘贴
写 BSP 驱动 2-3 周 最费时，每颗芯片一个驱动
写应用层（UI + Func + Tasks） 2-3 周 反复调
写 BootLoader 2-3 天 ST 官方代码 + 组装
调试 + 修 bug 2-4 周 最不可预测

总计：从零开始大概 1.5-2 个月（全职），大块时间在写 BSP 驱动和 UI 调试上。

从零构建 vs 复现

如果只是复现（硬件已有、代码已有）：

  1. CubeMX 两步打开 .ioc → 不改任何配置，点 Generate Code 重新生成一次 Core/
  2. Keil 打开 .uvprojx → 编译 → ST-Link 烧录
  3. 结束

你现在的学习路线就是先复现跑通，再逐步理解每一层怎么写的。
