第一步：搭建编译环境（PC）

1. 启用 WSL2

Windows PowerShell（管理员）：

1

wsl --install -d Ubuntu-22.04

重启电脑，进 Ubuntu 终端。

2. 安装依赖 

1
2

sudo apt update
sudo apt install build-essential cmake libsdl2-dev adb

3. 安装交叉编译工具链

工具链路径记录在 build.sh 里——arm-openwrt-linux-gcc。把这个工具链放到你电脑上，记住路径，改 build.sh 里的 toolchain_path。

第二步：在 PC 上跑模拟器（先验证 UI）

不需要板子，先在 PC 上把 UI 跑起来：

1
2
3

cd /mnt/c/Users/13403/Desktop/quanzhi-LVGL/app_sdk_v2/app_sdk
./build.sh -linux
./build/app/demo

此时应弹出一个 SDL2 窗口，显示智能桌面助手的 GUI。你可以用鼠标点击交互。

如果这里跑不通，检查 SDL2 是否装好。如果成功，说明 UI 代码没问题，可以继续。

第三步：交叉编译 T113 版本

1. 确认工具链

打开 build.sh，确认 toolchain_path 指向你电脑上交叉编译器的 bin 目录：

1

toolchain_path="/你的路径/toolchain/bin"

确保该目录下有 arm-openwrt-linux-gcc。

2. 编译 

1

./build.sh -t113

编译成功后产物在 build/app/demo（ARM 可执行文件）。

第四步：部署到板子

1. 连接板子

USB 线连接 PC 和 T113 板子。确认 ADB 识别到设备：

1

adb devices

应列出一个设备。

2. 推送文件 

1
2
3
4
5
6
7
8

# 第一次：推库文件（以后不改库就不需要重复这步）
adb push platform/t113/lib/* /usr/lib/

# 第一次：推资源文件（图片、字体、音频）
adb push build/app/res/* /usr/res/

# 每次改代码后：推可执行文件
adb push build/app/demo /usr/bin/

3. 在板子上跑 

1
2

adb shell
/usr/bin/demo &

板子的 LCD 屏幕应显示 GUI 界面。

第五步：设置开机自启 

1
2

adb shell
vi /etc/init.d/rc.final

在文件末尾加一行：

1

/usr/bin/demo &

保存退出，执行：

1

reboot

板子重启后自动启动 demo，上电即显示 GUI。

附录 B：常见问题

Q: build.sh 编译报错 “toolchain not found”

检查 build.sh 里的 toolchain_path，确认路径下有 arm-openwrt-linux-gcc。

Q: ADB 找不到设备

  • 检查 USB 线是否支持数据传输（不是只能充电的线）
  • 确认板子已开机
  • 在 WSL 里可能需要 sudo adb kill-server && sudo adb start-server

Q: 推上去跑了但是屏幕没显示

  • adb shell 进板子，看 ps | grep demo 确认进程在跑
  • 看有没有 printf 错误输出
  • 检查是否为横竖屏参数问题，可能需要 ./build.sh -clean 后重新 cmake

Q: 资源文件（图片/字体）显示不出来

检查 PROJECT_RES_URL 路径。T113 上资源路径是 /usr/res/，确认文件已推到这个目录。

Q: 编译好的 demo 很大

正常。因为 FreeType 和 LVGL 都是静态链接的，最终二进制约 2-3MB。
